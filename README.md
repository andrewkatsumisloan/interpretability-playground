# Exploring Interpretability

Small TransformerLens experiments for learning mechanistic interpretability.

## Layout

The repo is split between runnable examples and reusable support code:

- `scripts/01_logit_attribution.py` decomposes the residual stream and
  measures direct attribution to the `" Paris"` answer logit.
- `scripts/02_interpretability_basics.py` runs a clean/corrupted prompt pair,
  inspects predictions and cached activations, then performs a residual-stream
  activation patching sweep.
- `src/mechanistic_interpretability/` contains reusable utilities shared by
  examples: prompt constants, device selection, model loading, token helpers,
  prediction printing, and output-directory setup.

Going forward, `scripts/` should stay thin and example-oriented. If a helper is
useful across examples, put it under `src/mechanistic_interpretability/`. If a
file is generated by an experiment, keep it under ignored `outputs/`.

The older smoke test and first patching script were folded into the package
utility layer and the configurable basics script.

## Running

```bash
uv run python scripts/01_logit_attribution.py
uv run python scripts/02_interpretability_basics.py
```

Useful options:

```bash
uv run python scripts/02_interpretability_basics.py --model gpt2-small
uv run python scripts/02_interpretability_basics.py --device cpu
uv run python scripts/02_interpretability_basics.py --hook resid_post
```

## Generated Assets

Generated artifacts live under `outputs/`, which is ignored by git:

- `outputs/tables/` for CSV tables.
- `outputs/figures/` for plots and image outputs.
- `outputs/runs/` for JSON run metadata.

Output filenames include the model name and experiment type so multiple runs
can coexist, for example `gpt2-small_france_italy_resid_post_patch.csv`.

If the project later needs checked-in source assets, use a separate `assets/`
directory and keep it limited to hand-authored inputs, reference images, or
small fixtures. Large model outputs and regenerated plots should stay in
`outputs/`.
